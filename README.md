## LW-CDEV简介
---

## 设计理念：
 lw-cdev 是一款专为裸机、RTOS 场景设计的轻量级 I/O 设备框架，核心设计原则为极致轻量化与高可移植性。它针对单片机产品开发中常见的代码架构混乱、外围模块变更频繁、应用与驱动强耦合的痛点，提供标准化的解耦方案，实现应用层与底层驱动的彻底分离。框架资源占用极低，可无缝适配裸机（单线程）与 RTOS（多线程）开发场景。  

## 设备框架：
lw-cdev 参考 Linux 设备驱动的设计思想，打造了一套极简的 I/O 设备管理框架，支持快速封装标准化的设备接口，实现对 I/O 设备、字符设备、自定义时序 / 协议的统一管控。

框架采用分层设计：

+ 驱动层：由用户根据硬件特性实现底层接口，完成硬件寄存器、总线协议的适配；
+ 应用层：通过框架获取对应设备的句柄，借助标准化接口完成数据交互与设备控制，无需感知底层硬件与协议的实现细节。

基于这套架构，外围电路更换、模块升级、协议优化等变更，仅需调整驱动层的底层实现，完全不会影响应用层代码，避免应用层对硬件 / 协议的强依赖问题。相较于常规的结构化开发，它仅需前期增加少量的抽象适配工作，即可大幅降低产品全生命周期的迭代维护成本（常见于外围电路变更，通信模块变更，自定义协议变更等）。

## 支持设备：
该框架为通用框架,允许移植在任何C语言开发环境中。

## 建议与注意事项：
+ 建议绝大部分单片机上移植时，开启`lw_cdev_auto_locking`宏
+ 默认允许设备总线至多挂载128个设备，该数量为内存优化算法的最佳数量，若要扩增设备，需修改源码。
+ 文件中文注释格式默认为GB2312
